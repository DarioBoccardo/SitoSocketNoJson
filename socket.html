Introduzione alle Socket

Cos'è una Socket?

Una socket è un'interfaccia software che consente la comunicazione tra due dispositivi attraverso una rete. Le socket sono utilizzate per inviare e ricevere dati su reti di computer, come Internet. Ogni socket è identificata da un indirizzo IP e una porta specifica, che insieme definiscono un "endpoint" di comunicazione.

Le socket possono essere utilizzate per vari scopi, inclusi i trasferimenti di file, la messaggistica istantanea e la navigazione web. Esse operano tipicamente in due modalità: streaming o datagramma, a seconda del protocollo di rete scelto (TCP o UDP).

Tipologie di Socket

Le socket si classificano in due categorie principali, che sono legate ai protocolli di rete utilizzati per la comunicazione:

1. Stream Socket (TCP)

Le socket di tipo stream utilizzano il protocollo di comunicazione TCP (Transmission Control Protocol), che garantisce una trasmissione affidabile dei dati. Con una socket di tipo stream, i dati vengono inviati in una sequenza continua e devono essere ricevuti nello stesso ordine in cui sono stati inviati. TCP assicura che:

  • I dati arrivino senza errori.
  • Vengano ricevuti in ordine.
  • Non ci siano duplicazioni di dati.
  • Se un pacchetto di dati viene perso, venga ritrasmesso.

Le applicazioni che richiedono alta affidabilità, come il web (HTTP) o la posta elettronica (SMTP), utilizzano spesso questo tipo di socket.

2. Datagram Socket (UDP)

Le socket di tipo datagramma utilizzano il protocollo UDP (User Datagram Protocol), che è più veloce ma meno affidabile rispetto al TCP. UDP invia i pacchetti di dati senza garantire che arrivino correttamente, né in ordine né senza duplicazioni. Questo lo rende ideale per applicazioni in tempo reale come il voip (Voice over IP) o lo streaming video, dove la velocità è più importante della perfetta affidabilità.

UDP è utilizzato in scenari dove la perdita di alcuni pacchetti non è critica e la velocità di trasmissione deve essere prioritaria. Ad esempio:

  • Streaming audio e video (es. YouTube, Netflix)
  • Videogiochi online
  • Chiamate VoIP

Processo di Comunicazione con le Socket

Una comunicazione tra due dispositivi tramite socket avviene in più fasi, che coinvolgono sia il server che il client. Ecco un flusso tipico di comunicazione:

Fasi di Comunicazione

  1. Creazione della Socket: Prima di tutto, sia il client che il server creano una socket. Il server crea una socket in ascolto su una porta specifica e su un indirizzo IP, pronto ad accettare le connessioni in arrivo.
  2. Connessione Client-Server: Il client stabilisce una connessione con il server utilizzando l'indirizzo IP e la porta del server. Per i socket di tipo stream (TCP), questa connessione è orientata alla connessione, e il server accetta o rifiuta la connessione a seconda della disponibilità.
  3. Comunicazione: Una volta stabilita la connessione, il client e il server possono scambiarsi dati. Il server ascolta continuamente le richieste dei client, mentre i client inviano richieste al server e attendono risposte.
  4. Chiusura della Connessione: Quando la comunicazione è terminata, entrambe le parti chiudono la connessione, liberando le risorse. Il client e il server inviano messaggi di chiusura per segnare la fine della sessione.

Alcuni Esempi di Utilizzo delle Socket

Le socket sono utilizzate in molti scenari di rete. Ecco alcuni esempi comuni di applicazione:

  • HTTP: Quando un browser web si connette a un server per caricare una pagina web, utilizza una socket per la comunicazione tra il client e il server.
  • FTP: Il File Transfer Protocol (FTP) utilizza le socket per trasferire file tra client e server su una rete.
  • Telnet: Le socket sono utilizzate anche in Telnet, un protocollo che permette di accedere a un computer remoto attraverso la rete.
  • Chiamate VoIP: I software di comunicazione come Skype utilizzano le socket per inviare pacchetti di voce in tempo reale tramite UDP.
